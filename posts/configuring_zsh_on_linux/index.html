Configuring ZSH on Linux

Jordan Williams included in Systems
 2020-09-19  607 words   3 minutes 
Contents

  1. Install ZSH.[1]

    $ sudo dnf install -y zsh

  2. Add /usr/bin/zsh to /etc/shells.[2]

    $ command -v zsh | sudo tee -a /etc/shells
/usr/bin/zsh

  3. Set the default shell to zsh.

    $ chsh -s `which zsh`
Changing shell for jordan.
Password:
Shell changed.
  4. Now, open up a fresh terminal session to start using zsh.

oh-my-zsh

  1. If you don’t already have git install it in order to retrieve oh-my-zsh.

    $ sudo dnf install -y git

  2. Install the oh-my-zsh git repository.

    $ git clone https://github.com/ohmyzsh/ohmyzsh.git ~/.oh-my-zsh

  3. Copy over the zshrc template file.

    Caution:

    You may want to backup your existing .zshrc before continuing.

    $ cp ~/.oh-my-zsh/templates/zshrc.zsh-template ~/.zshrc

  4. Load the new .zshrc configuration file.

    $ source ~/.zshrc

Antigen

[3]

  1. Download the Antigen repository for CentOS 7.footnote[https://software.opensuse.org/download.html?project=shells%3Azsh-users%3Aantigen&package=antigen]

    $ wget https://download.opensuse.org/repositories/shells:zsh-users:antigen/CentOS_7/shells:zsh-users:antigen.repo

  2. Place the repository in the /etc/yum.repos.d/ directory.

    $ sudo mv shells:zsh-users:antigen.repo /etc/yum.repos.d

  3. Install Antigen.

    $ sudo dnf install -y antigen

  4. Create the .antigenrc configuration file.footnote:[]

  5. Load the Antigen configuration file from the .zshrc file.

System-level Update Service

CentOS 7 does not support systemd user services, which seem like a good fit for updating your Antigen plugins on a schedule. It is still possible to create a system-level unit, though, which is what these instructions describe.

  1. Add a systemd service to update the Antigen plugins.

    This systemd unit executes the antigen update command after loading the .zshrc for the user jordan.[4] This is run as the user jordan and it requires the network be online.

  2. Verify the network-online.target exists.[5]

    $ systemctl is-enabled NetworkManager-wait-online.service
enabled

  3. Test run the new systemd unit.

    $ sudo systemctl start update-jordan-antigen.service

  4. Check the output of the command to make sure everything worked.

    $ sudo systemctl status update-jordan-antigen.service

User-level Update Service

$ mkdir -p ~/.config/systemd/user

  1. Add a systemd service to update the Antigen plugins.[6]

  2. Add a systemd timer to update the Antigen plugins once per week.

  3. Start the Antigen update timer upon login.[7]

    $ systemctl --user enable update-antigen.timer
